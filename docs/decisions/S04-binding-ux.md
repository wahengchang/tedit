# 決議 S04(D16):變數綁定操作 UX = 屬性面板開關 + 畫布角標

> 日期:2026-06-12。依交接方傾向定案,與 S03(獨立 bindings 區塊)互為表裡。

## 裁決

綁定操作走**屬性面板開關**,已綁元素在畫布上顯示**角標**;不採文字內打 `{{}}`。

## 線框

```
┌─畫布──────────────────────────┐  ┌─屬性面板(選中 txt1)─────────┐
│                               │  │ 內容    [春江潮水連海平…    ] │
│   ┌────────────┐{t}           │  │ 字體    [Noto Sans TC   ▾] │
│   │ 春江潮水連海平… │←角標       │  │ 字級    [28] 色 [#f4f1ea]  │
│   │ (已綁 title)  │            │  │ ──────────────────────── │
│   └────────────┘              │  │ ◉ 綁定變數                 │
│   ┌──────┐                    │  │   變數名 [title        ]   │
│   │ 圖片  │{p}                 │  │   (取消綁定 = 關閉開關)      │
│   └──────┘                    │  └───────────────────────────┘
└───────────────────────────────┘
```

- 開關開啟 → 輸入變數名 → bindings 區塊新增/更新一筆;關閉 → 移除該筆。
- 角標(如 `{t}`)畫在元素右上角,**僅編輯器 UI 層,不進 engine 場景**,
  存檔與 headless 出圖不受影響(同像素不破)。
- 同名變數綁多處:面板輸入同名即可,vars 列表自然聚合(S03 天然支援)。

## 依據

1. 與 S03 配套:綁定是 bindings 表的一筆記錄,面板開關正是這筆記錄的 1:1 UI,
   不需要解析文字、不需要跳脫規則。
2. 設計時預設值與綁定分離:文字內容欄位永遠是「現在畫布上看到的字」,
   不會出現 `{{title}}` 佔位字撐壞版面的問題。
3. 純展示性角標可在 M4 後期再加,不擋核心流程。
